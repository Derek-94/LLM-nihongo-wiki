# AGENTS.md — 위키 운영 규칙 (모든 AI 에이전트 공통)

이 저장소를 다루는 모든 에이전트(OpenClaw/OpenAI, Claude Code 등)는 이 규칙을 따른다.
이 파일이 두 모델의 일관성을 보장하는 "계약서"다. 설계 배경은 [DESIGN.md](DESIGN.md) 참고.

## 공통 원칙
- 모든 문서는 **plain markdown**. 자체 DB·바이너리 금지.
- 페이지 간 링크는 **일반 markdown 링크** `[텍스트](상대경로.md)` 사용.
  `[[wikilink]]` 금지 (GitHub에서 렌더링·이동 안 됨).
- 일본어는 가능하면 **한자(읽기)** 병기: 例) 開ける(あける).
- 변경 후에는 의미 있는 단위로 `git commit` → `git push`.

## 폴더 구조
```
inbox.md            ← 캡처된 원본(raw)이 시간순으로 쌓임. 정리 후 비워짐.
mistakes.md         ← 틀린 것 누적. 시험 출제의 1순위 소스.
srs.md              ← 단어/문법별 복습 스케줄(간격 반복 메타데이터).
grammar/_index.md   ← 문법 전체 지도
grammar/*.md        ← 문법 주제별 페이지
vocab/_index.md     ← 단어 지도
vocab/*.md          ← 분야/품사별 단어 페이지
```

## 역할별 규칙

### 1) 캡처 에이전트 (OpenClaw / 미니PC)
- 텔레그램으로 들어온 내용을 **`inbox.md` 맨 아래에 append만** 한다.
- 형식: `## YYYY-MM-DD HH:MM` 헤더 + 받은 내용 그대로(가공 금지).
- **위키 본문(grammar/·vocab/)은 직접 수정하지 않는다.** 정리는 종합 에이전트 몫.
- append 후 `git commit -m "inbox: ..." && git push`.

### 2) 종합 에이전트 (Claude Code / 맥, 야간 배치)
- `git pull` 후 `inbox.md`를 읽어 내용을 해당 페이지로 라우팅:
  - 문법 → `grammar/주제.md` 갱신, 관련 페이지끼리 링크 연결
  - 단어 → `vocab/*.md`에 추가, `srs.md`에 신규 항목 등록
- **모순/중복 점검**: 기존 내용과 충돌하면 덮어쓰지 말고 비교·통합.
- 정리 끝난 inbox 항목은 제거(또는 `## 처리완료`로 이동).
- 변경 요약을 commit 메시지에 남긴다. 예: `vocab: 5/21 단어 8개 / fix: 조사 に·で 혼동 기록`.

### 3) 시험 에이전트 (OpenClaw / 텔레그램)
- 출제 우선순위: `mistakes.md`(자주 틀린 것) > `srs.md`상 오늘 복습 예정 항목 > 최근 학습.
- 문법은 **문장 생산형**으로 출제하고 **조사까지 채점**한다.
- 오답은 `mistakes.md`에 `날짜 · 문제 · 내 답 · 정답 · 짧은 해설`로 기록.
- 정/오답에 따라 `srs.md`의 `다음복습일`을 갱신(맞으면 간격↑, 틀리면 당김).

## srs.md 항목 형식 (예시)
```
| 항목 | 종류 | 마지막테스트 | 틀린횟수 | 다음복습일 |
|------|------|-------------|---------|-----------|
| あまり | vocab | 2026-05-21 | 1 | 2026-05-23 |
| 사역 조사 が→に | grammar | 2026-05-21 | 2 | 2026-05-22 |
```
